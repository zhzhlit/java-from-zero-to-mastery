# 小项目

小项目用于集中训练某一个知识点或某一阶段能力。每个小项目都应保持范围清晰、可独立运行、可独立验收，并能回链到对应课程或路线。

## 小项目验收清单

每个小项目提交前必须补齐以下内容：

- **聚焦目标**：说明本项目训练的核心能力、前置知识和明确非目标，避免把阶段练习扩展成大型系统。
- **运行命令**：提供从仓库根目录或项目目录执行的完整命令，包括构建、启动和必要的参数说明。
- **测试**：提供自动化测试命令；如果暂时只能手工验证，必须说明手工步骤和后续补测试计划。
- **预期输出**：给出关键命令、接口或页面的预期结果，便于学习者判断是否运行成功。
- **学习引用**：链接到相关课程、手册、路线图或主项目阶段，说明学习者完成后应回到哪里继续学习。

## 推荐 README 结构

```markdown
# 项目名称

## 聚焦目标

## 前置知识

## 运行命令

## 测试

## 预期输出

## 学习引用
```

小项目可以根据需要补充“常见问题”“扩展挑战”或“实现提示”，但不能省略验收清单中的核心项。
