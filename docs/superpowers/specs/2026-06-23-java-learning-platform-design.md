# Java 从入门到精通开源项目设计

## 1. 项目概述

本项目面向大学生、Java 初级开发者和 Java 中级开发者，建设一套可系统学习、可快速查阅、可动手实践的开源 Java 知识体系。

项目不定位为单纯的电子书。它由三条相互关联的内容线组成：

1. 系统课程：帮助学习者按照明确路线从零基础成长为具备企业开发能力的 Java 工程师。
2. 精选手册：帮助在职开发者快速查询 API、原理、实践边界和故障排查方法。
3. 实战项目：通过独立小项目和一个持续演进的主项目，把知识转化为工程能力。

项目首发内容使用中文，保留英文站点的目录和国际化配置，但第一阶段不维护双语内容。

## 2. 目标与非目标

### 2.1 项目目标

- 建立从 Java 零基础到中高级工程能力的完整学习路线。
- 为重要知识点提供可使用 Java 21 编译和运行的代码示例。
- 通过练习、自动测试和项目任务帮助学习者验证学习效果。
- 为开发者提供适合日常工作的精选速查内容。
- 通过统一规范、自动化检查和版本计划，使项目可以长期维护。
- 提供适合社区参与的 Issue、Pull Request 和内容贡献流程。

### 2.2 第一阶段非目标

- 不在首个版本中写完所有课程。
- 不同时完整维护 Java 8、Java 17 和 Java 21 三套实现。
- 不把前端、大数据和 AI 扩展为独立完整课程体系。
- 不为每篇课程机械复制一份速查文档。
- 不在项目尚未成熟时拆分大量子仓库。
- 不以覆盖尽可能多的框架为目标，优先讲清稳定、通用的工程能力。

## 3. 受众与使用方式

### 3.1 大学生和零基础学习者

按照学习路线依次阅读系统课程，运行示例，完成分级练习和阶段项目。每章明确前置知识、学习目标和后续路线，避免学习者在大量技术名词之间迷失。

### 3.2 Java 初级开发者

按主题补齐集合、并发、JVM、数据库、Web、Spring 等知识，通过主项目理解企业项目中的组合应用。

### 3.3 Java 中级开发者

使用速查手册、原理章节、故障排查清单、性能专题和源码阅读内容解决实际问题，也可以参与内容审核和项目贡献。

## 4. 技术基线

- Java：Java 21 LTS
- 构建工具：Maven
- 企业开发：Spring Boot 3
- 测试：JUnit 5
- 文档：Markdown
- 文档站：VitePress
- 发布：GitHub Pages
- 外部依赖环境：Docker Compose
- 持续集成：GitHub Actions

Java 8 和 Java 17 的差异通过迁移专题讲解，不要求所有示例同时兼容多个 Java 版本。

## 5. 仓库策略

采用“核心单仓库，成熟项目按需拆分”的策略。

第一阶段将课程、手册、示例、练习、小项目和主项目统一放在一个仓库中，为学习者提供单一入口。后续当主项目或某个小项目具备独立的发布节奏、社区和维护需求时，可以拆分为单独仓库；本仓库继续保留教学文档、精简代码和稳定链接。

初始目录规划如下：

```text
java-from-zero-to-mastery/
├── README.md
├── docs/
│   ├── zh-CN/
│   │   ├── guide/
│   │   ├── handbook/
│   │   ├── projects/
│   │   ├── interview/
│   │   └── roadmap/
│   ├── en/
│   └── superpowers/specs/
├── examples/
├── exercises/
├── mini-projects/
├── flagship-project/
├── website/
└── .github/
```

目录职责：

- `docs/zh-CN/guide`：系统课程正文。
- `docs/zh-CN/handbook`：精选速查专题，不与课程逐篇重复。
- `docs/zh-CN/projects`：小项目和主项目的教学说明。
- `docs/zh-CN/interview`：复习清单、面试题和答案解析。
- `docs/zh-CN/roadmap`：学习路线、阶段目标和版本进度。
- `docs/en`：未来英文内容入口；第一阶段仅保留必要说明，不创建空章节树。
- `examples`：与知识点对应、可独立运行的最小示例。
- `exercises`：练习题、测试用例和参考答案。
- `mini-projects`：用于训练单项或阶段能力的独立项目。
- `flagship-project`：贯穿课程的在线学习与知识社区项目。
- `website`：VitePress 配置、主题和文档站构建代码。
- `.github`：CI、GitHub Pages、Issue 模板、PR 模板和贡献配置。

## 6. 内容架构

### 6.1 十阶段学习路线

1. 开发准备：JDK 21、IDEA、Maven、Git、命令行和调试方法。
2. Java 基础：语法、流程控制、数组、方法、面向对象和异常。
3. Java 核心：集合、泛型、反射、注解、IO/NIO、日期时间和函数式编程。
4. 计算机与工程基础：数据结构、算法、操作系统、网络、设计模式、测试和代码规范。
5. 数据库与 Web：MySQL、SQL、事务、JDBC、HTTP、Servlet 和 REST。
6. 企业级开发：Spring、Spring MVC、Spring Boot、MyBatis、JPA、权限和接口设计。
7. 中间件与分布式：Redis、消息队列、Elasticsearch、任务调度和分布式事务。
8. Java 高级：并发编程、JVM、性能分析、故障排查和源码阅读。
9. 微服务与云原生：Spring Cloud、Docker、Kubernetes、CI/CD 和可观测性。
10. 扩展方向：Vue/TypeScript 基础、大数据、响应式编程、Spring AI 和 LLM 应用。

前端、大数据和 AI 是 Java 工程师的扩展能力，不改变 Java 后端作为项目主线的定位。

### 6.2 每阶段的交付物

- 系统课程
- 可运行示例
- 基础、进阶、挑战三级练习
- 参考答案及自动测试
- 一个阶段小项目或明确的项目任务
- 必要的精选速查卡
- 面试与复习清单
- 主项目演进任务

### 6.3 双轨内容

系统课程负责建立完整心智模型，速查手册负责快速解决工作问题。

课程内容重点包括问题背景、概念、原理、示例和练习。速查手册只选择高频且适合检索的主题，例如：

- 集合选型与时间复杂度
- 并发工具选择
- JVM 参数与故障现象
- Spring Boot 常用配置
- SQL 优化检查清单
- Redis 数据结构和典型用途
- HTTP 状态码及接口设计约定

速查手册引用系统课程中的深入解释，避免复制长篇正文。

## 7. 实战体系

### 7.1 小项目

小项目用于集中训练单项能力，保持规模可控，并提供独立的运行说明和验收测试。候选项目包括：

- 命令行学生管理系统：语法、面向对象和集合。
- 文件索引与搜索工具：IO/NIO、异常和并发。
- 简易聊天室：Socket、线程模型和协议设计。
- 任务调度器：线程池、并发控制和可观测性。
- 轻量依赖注入容器：反射、注解和设计模式。
- URL 短链服务：Spring Boot、数据库、缓存和接口设计。

具体小项目随对应阶段设计，不要求第一版一次性创建全部项目。

### 7.2 贯穿式主项目

主项目为“在线学习与知识社区”，其业务与本开源项目的学习场景一致。主项目按课程阶段逐步演进：

```text
命令行课程管理
→ Spring Boot 单体应用
→ 前后端分离
→ Redis 与搜索
→ 消息队列与异步任务
→ 高并发与性能优化
→ 微服务拆分
→ Docker 与 Kubernetes
→ AI 问答与学习助手
```

核心业务领域包括用户与权限、课程、文章、评论、收藏、搜索、通知、学习进度、订单或付费扩展、运营统计和 AI 学习助手。

每个阶段只引入支撑当前教学目标所需的业务，不提前建设完整平台。

## 8. 章节规范

每篇系统课程使用统一结构：

1. 学习目标
2. 前置知识
3. 真实问题场景
4. 核心概念
5. 原理与设计思想
6. 可运行代码
7. 常见错误与排查方法
8. 最佳实践及适用边界
9. 分级练习
10. 面试与复习题
11. 本章总结
12. 下一步学习建议

根据主题性质，可以合并不适用的小节，但不得省略学习目标、可验证示例或总结。

内容必须明确区分：

- Java 或相关标准的规范保证。
- 某个 JDK、框架或中间件版本的具体实现。
- 社区惯例和项目建议。

版本敏感内容必须标注适用版本和最后验证日期。

## 9. 示例、练习与答案

### 9.1 示例

- 示例必须服务于一个明确知识点。
- 基础示例优先保持最小化，不引入与主题无关的框架。
- 每个示例提供运行命令、预期结果和关键解释。
- Java 示例必须使用 Java 21 在 CI 中编译。
- Spring Boot 示例必须提供启动方式、最小配置和健康验证方法。

### 9.2 练习

- 基础练习验证概念和语法。
- 进阶练习要求组合多个知识点。
- 挑战练习接近真实工程问题，允许存在多种合理方案。
- 可自动验证的练习使用 JUnit 5 测试。
- 无法自动验证的设计题提供验收标准和评审要点。

### 9.3 答案

参考答案默认使用折叠区域或独立目录存放。答案不仅给出最终代码，还要解释思路、边界条件和常见错误。对于开放题，提供参考方案而不是宣称存在唯一答案。

## 10. 错误处理与学习者体验

- 示例中的失败场景必须说明预期异常，而不是只展示成功路径。
- 外部服务不可用时，运行文档要提供检查命令和常见排查步骤。
- 文档中的命令应说明执行目录和适用操作系统；存在差异时分别给出。
- 依赖数据库、缓存、消息队列或搜索服务的示例，应通过 Docker Compose 提供可复现环境。
- 主项目的错误响应、日志、校验和异常处理随着课程阶段逐步演进，避免第一阶段引入过度复杂的统一框架。
- 文档站应提供学习路线入口、按主题导航、站内搜索和上一章/下一章链接。

## 11. 测试与质量保障

CI 至少执行以下检查：

- Java 21 编译
- Maven 单元测试和必要的集成测试
- VitePress 文档构建
- Markdown 内部链接检查
- 格式和基础静态检查

对于依赖外部服务的集成测试，优先使用容器化测试环境；耗时较长的完整测试可以与基础 PR 检查分层运行。

内容状态统一使用：

- `planned`：已规划，尚不可学习。
- `draft`：编写中，内容可能明显变化。
- `learning-ready`：内容完整，可供学习。
- `verified`：在声明的技术版本上完成复核和自动验证。

只有 `learning-ready` 和 `verified` 内容进入默认学习路线。

## 12. 发布与版本计划

- v0.1 项目骨架：路线图、目录、写作规范、VitePress、CI 和贡献指南。
- v0.2 Java 入门：开发环境、语法、面向对象、基础练习和命令行项目。
- v0.3 Java 核心：集合、泛型、异常、IO、函数式编程和 JUnit。
- v0.4 Web 开发：MySQL、HTTP、Spring Boot、MyBatis 和主项目单体版。
- v0.5 中高级：并发、JVM、Redis、消息队列、性能和故障排查。
- v0.6 分布式：微服务、搜索、分布式事务和可观测性。
- v0.7 云原生：Docker、Kubernetes 和 CI/CD。
- v0.8 扩展能力：前端、大数据和 AI 应用。
- v1.0 正式版：主路线完整、关键示例验证通过、文档站稳定。

版本号表示项目整体成熟度，不强制与某个依赖版本绑定。

## 13. 社区与维护

- 使用 GitHub Issues 管理勘误、内容需求和缺失主题。
- 使用 GitHub Projects 跟踪章节和版本状态。
- Pull Request 必须通过文档构建、代码编译和自动测试。
- 提供内容贡献、代码贡献、勘误和安全问题模板。
- 每季度检查核心依赖版本、失效链接和被标记为 `verified` 的重点内容。
- 对版本升级采用专题迁移和分批验证，不直接全仓机械替换。

建议许可证：

- 代码：Apache License 2.0。
- 文档：Creative Commons Attribution 4.0 International。

最终采用许可证前，需要在 README 和贡献指南中明确代码片段、文档内容和第三方素材各自适用的授权范围。

## 14. v0.1 验收标准

v0.1 完成时应满足：

- 仓库目录和模块职责清晰。
- README 能说明项目定位、受众、路线、运行方式和贡献入口。
- VitePress 能在本地构建并预留未来英文版配置。
- GitHub Pages 工作流具备可部署配置。
- Java 21 Maven 聚合工程能够编译和运行测试。
- 至少提供一篇符合模板的示范课程。
- 至少提供一个可运行示例和一组带 JUnit 5 的示范练习。
- 提供主项目演进路线，但只实现当前版本需要的最小骨架。
- 提供写作规范、贡献指南、行为准则、Issue/PR 模板和许可证说明。
- CI 能发现代码编译失败、测试失败、文档构建失败和内部链接错误。

## 15. 主要风险与应对

### 范围过大

以版本计划和内容状态控制范围。默认导航只展示已经达到可学习状态的内容，避免大量空目录造成“项目很大但无法使用”的观感。

### 内容重复

课程讲完整过程，手册提供结论和检查清单，并通过链接复用深入解释。

### 示例失效

将关键示例纳入 Maven 聚合构建和 CI；版本敏感内容记录最后验证日期。

### 维护者精力不足

优先完成一条纵向可学习路径，再横向扩展主题。新技术进入主路线前必须证明其长期价值。

### 初学者认知负担过重

每章声明前置知识，严格控制示例依赖，并让主项目随课程逐步演进。

## 16. 后续工作

设计通过后，下一步编写 v0.1 的详细实施计划。实施计划应将目录初始化、Maven 工程、VitePress、CI、内容模板、示范课程、示范练习和社区文件拆分为可独立验证的小任务。
