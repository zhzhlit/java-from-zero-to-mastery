# v0.2.1 Java 入门基础设计

## 1. 版本目标

v0.2.1 建设第一段可连续学习的 Java 入门路线，使零基础学习者能够完成开发环境搭建，并掌握变量与数据类型、流程控制和方法。

本版本以 IntelliJ IDEA 为主要教学工具，以 Maven 命令作为跨平台、跨编辑器和 CI 环境中的统一验证标准。

## 2. 目标受众

- 第一次系统学习 Java 的大学生。
- 了解少量编程概念但 Java 基础不稳的学习者。
- 需要复习 Java 基础语法和工程习惯的初级开发者。

学习者不需要提前掌握 Maven、JUnit 或 IntelliJ IDEA。

## 3. 范围

### 3.1 开发环境与 IntelliJ IDEA

- 安装和验证 JDK 21。
- 区分 JDK、JVM 和 JRE 的职责。
- 使用 IntelliJ IDEA 导入 Maven 聚合项目。
- 在 IDEA 中运行程序和 JUnit 测试。
- 使用断点、单步执行、变量视图和调用栈进行基础调试。
- 使用 Maven 命令验证模块，不依赖 IDEA 私有配置。
- 介绍常见环境问题：Java 版本错误、项目 SDK 缺失、Maven 未同步和工作目录错误。

### 3.2 变量与数据类型

- 变量声明、赋值、初始化和作用域。
- 八种基本数据类型和引用类型的入门概念。
- 字面量、常量和 `var` 的适用边界。
- 自动类型提升、显式类型转换和溢出风险。
- 算术、比较、逻辑、赋值和条件运算符。
- 整数除法、浮点精度和金额不使用 `double` 的原因。
- 命名规范、避免魔法数字和使用有意义变量名。

### 3.3 流程控制

- `if`、`else if` 和 `else`。
- 传统 `switch` 与 Java 21 可用的 `switch` 表达式。
- `for`、增强 `for`、`while` 和 `do-while`。
- `break`、`continue` 和提前返回。
- 循环边界、无限循环、分支遗漏等常见错误。
- 使用卫语句降低嵌套，控制条件复杂度。

### 3.4 方法

- 方法声明、调用、参数和返回值。
- `void` 方法和有返回值方法的使用场景。
- 方法重载及其解析边界。
- Java 只有值传递，包括对象引用值的传递。
- 局部变量、参数和作用域。
- 将长流程拆分为职责单一的方法。
- 参数过多、布尔参数、重复代码和超长方法等基础坏味道。

## 4. 内容深度

每章采用四层内容：

1. 基础语法：帮助学习者正确编写和运行代码。
2. 必要原理：解释能避免常见误解的语言规则。
3. 工程实践：展示在真实项目中如何组织和约束代码。
4. 面试与复习：帮助学习者准确表达概念和取舍。

字节码、栈帧、JIT、HotSpot 源码和 JVM 深入实现不在正文展开。确有帮助时，可以使用“深入阅读”提示链接到未来高级章节，但不得打断基础学习主线。

## 5. 课程交付物

每章至少包含：

- 一篇符合项目写作规范的完整课程。
- 2 至 3 个聚焦单一知识点的可运行示例。
- 基础、进阶、挑战三级练习。
- 可自动验证的 JUnit 5 测试。
- 常见错误、症状、原因和排查方式。
- 5 至 8 道面试与复习题。
- IDEA 操作说明和 Maven 验证命令。
- 上一章、下一章和相关练习入口。

“开发环境与 IDEA”章节可以不提供三个 Java 示例，但必须提供可重复执行的环境检查、项目导入、运行、测试和调试步骤。

## 6. 示例代码结构

四章共用一个 `java-basics` Maven 模块，按主题分包：

```text
examples/java-basics/
├── pom.xml
└── src/
    ├── main/java/io/github/javamastery/basics/
    │   ├── variables/
    │   ├── controlflow/
    │   └── methods/
    └── test/java/io/github/javamastery/basics/
        ├── variables/
        ├── controlflow/
        └── methods/
```

组织原则：

- 每个示例类只解释一个主要概念。
- 示例类可以包含 `main` 方法，核心逻辑应提取为可测试的方法。
- 测试与生产代码使用对应包结构。
- 示例不依赖第三方业务框架。
- 所有示例通过 Java 21 编译和 JUnit 5 测试。

## 7. 练习与答案结构

练习采用 `starter` 与 `solution` 双目录：

```text
exercises/java-basics/
├── pom.xml
├── README.md
├── starter/
│   ├── pom.xml
│   └── src/
└── solution/
    ├── pom.xml
    └── src/
```

### 7.1 Starter

- 提供题目说明、方法签名、必要代码骨架和学习者测试入口。
- 代码必须能够编译。
- 未实现逻辑使用明确且可运行的占位行为，不使用会导致仓库 CI 失败的永久失败测试。
- 学习者可以运行单独命令检查自己的实现。
- 不在注释中直接泄露完整答案。

### 7.2 Solution

- 提供完整参考实现和测试。
- 所有测试必须在 CI 中通过。
- README 或代码注释解释思路、边界条件和常见错误。
- 开放题可以提供一种参考方案，不宣称唯一正确。

### 7.3 CI 策略

- 默认 Maven reactor 编译 `starter`，验证骨架没有语法或依赖问题。
- 默认 Maven reactor完整测试 `solution`。
- 学习者测试可以通过 Maven profile 或明确的模块命令启用。
- 仓库默认分支不得因为故意未完成的练习持续失败。

## 9. IDEA 与命令行教学原则

- IDEA 用于日常编辑、运行、测试、调试和项目浏览。
- Maven 命令用于标准化验证和问题排查。
- 不提交 `.idea`、`*.iml` 或依赖本机路径的配置。
- 每个操作先给出 IDEA 路径，再给出等价 Maven 命令。
- IDEA 截图不是完成操作的唯一依据，关键结果必须可以用文本或命令验证。
- Windows、macOS 和 Linux 存在差异时，明确标注命令或路径差异。

## 10. 测试与验收

v0.2.1 完成时必须满足：

- 4 篇新增课程达到 `learning-ready` 或 `verified`。
- `examples/java-basics` 所有示例编译并测试通过。
- `exercises/java-basics/solution` 所有测试通过。
- `exercises/java-basics/starter` 在默认状态下能够编译。
- IDEA 导入、运行、测试和调试步骤经过实际验证。
- 每章的示例命令、练习命令和预期结果一致。
- 文档内部链接检查通过。
- VitePress 构建通过。
- Maven 全仓 `verify` 通过。
- GitHub CI 通过，合并后文档站自动部署。

## 12. 非目标

v0.2.1 不包含：

- 数组和字符串的系统课程。
- 面向对象、异常和集合课程。
- 命令行学生管理系统。
- JVM 字节码、栈帧、JIT 或源码深入分析。
- Java 8 和 Java 17 的兼容实现。
- 英文课程翻译。
- VS Code 或 Eclipse 的平行完整教学路线。

## 13. 风险控制

### 初学内容过深

以“能正确编写、解释和排查”为边界。底层实现只提供后续阅读入口。

### 示例模块变成杂物箱

按主题分包，每个类只承担一个教学目标；新增内容必须归属明确章节。

### Starter 导致 CI 失败

永久测试只验证可运行骨架；要求学习者完成后才能通过的测试使用显式命令或 profile 启用。

### IDEA 教程随版本变化

优先描述稳定操作概念，并同时提供 Maven 命令作为不依赖 UI 的验证路径。

### 课程与代码脱节

文档命令直接指向仓库模块；示例和答案进入默认 CI。
