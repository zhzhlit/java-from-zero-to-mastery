# v0.2.2 数组、字符串与调试设计

## 1. 版本目标

v0.2.2 延续 v0.2.1 的 Java 入门路线，补齐进入面向对象之前必须掌握的数组、字符串和基础调试能力。

学习者完成本版本后，应能使用数组保存和统计一组数据，正确处理常见字符串操作，并使用 IntelliJ IDEA 调试器定位循环边界、数组越界和字符串比较问题。综合练习以“成绩统计器”为主线，把三个主题连接成一次完整实践。

## 2. 目标受众与前置知识

目标受众保持不变：

- 正在系统学习 Java 的大学生。
- Java 基础不完整的初级开发者。
- 需要复习基础语法与工程习惯的开发者。

开始本版本前，学习者应完成 v0.2.1，能够：

- 使用 JDK 21 和 Maven 编译、测试项目。
- 编写变量、分支、循环和方法。
- 在 IntelliJ IDEA 中运行 Java 程序和 JUnit 测试。

本版本不要求提前掌握类设计、集合、异常处理或 Stream API。

## 3. 课程范围

### 3.1 第 6 课：数组

- 一维数组的声明、创建、初始化和默认值。
- 下标、长度和合法访问范围。
- 普通 `for` 与增强 `for` 的选择。
- 遍历、查找、求和、平均值、最大值和最小值。
- 数组作为方法参数和返回值。
- 引用语义、别名和防御性复制的入门概念。
- `Arrays` 常用能力：排序、比较、复制和文本展示。
- 空数组、`null`、数组越界和循环边界错误。
- 二维数组只讲声明、遍历和表格型数据，不深入多维算法。

### 3.2 第 7 课：字符串与 `StringBuilder`

- `String` 的创建、不可变性和字符串常量池的必要概念。
- `length`、`charAt`、`substring`、`contains`、`indexOf`、`replace`、`split`、`trim` 和 `strip`。
- 使用 `equals`、`equalsIgnoreCase` 比较内容，避免使用 `==`。
- 空字符串、空白字符串与 `null` 的区别。
- 字符串拼接及循环内反复拼接的问题。
- `StringBuilder` 的创建、追加、插入、删除和结果转换。
- 文本解析、规范化和报告生成。
- Unicode、字符和代码点只提示边界，不在本版本深入展开。
- 正则表达式只在 `split` 的转义注意事项中简要说明。

### 3.3 第 8 课：IntelliJ IDEA 调试

- Debug 与普通 Run 的区别。
- 设置、禁用和删除断点。
- Step Over、Step Into、Step Out 和 Resume Program。
- 查看 Variables、Watches 和 Call Stack。
- 条件断点与表达式求值的基础用法。
- 在 JUnit 测试中启动调试。
- 使用最小可复现输入定位问题。
- 根据异常类型、堆栈和变量状态形成排查假设。
- 修复数组越界、循环少执行或多执行一次、错误使用 `==` 比较字符串等典型问题。

调试课程独立成章。它复用数组、字符串和成绩统计器中的真实代码，不引入一套与学习路线脱节的演示程序。

## 4. 综合练习：成绩统计器

综合练习处理一组以字符串形式提供的学生成绩，并生成统计报告。

### 4.1 核心能力

学习者需要完成：

- 将逗号分隔的成绩文本解析为 `int[]`。
- 校验成绩必须位于 `0` 到 `100`。
- 计算人数、总分、平均分、最高分和最低分。
- 按给定阈值统计及格人数。
- 使用 `StringBuilder` 生成稳定、可测试的文本报告。
- 通过测试失败信息和 IDEA 调试器定位边界错误。

### 4.2 输入与边界

- 输入示例：`"88, 92, 76, 100"`。
- 分隔符两侧允许空白。
- `null`、空文本和只有空白的文本属于非法输入。
- 非整数、缺失项和超出 `0` 到 `100` 的成绩属于非法输入。
- 完整参考实现使用 `IllegalArgumentException` 表达调用方输入错误；课程只解释这一具体用法，不提前展开异常体系。
- 平均分使用 `double`，报告中的格式化规则必须固定，避免测试依赖本机区域设置。

### 4.3 非目标

成绩统计器不保存学生姓名，不使用 `Student` 类，不进行文件读写，也不实现交互式菜单。这些能力留给面向对象、I/O 和项目实战章节。

## 5. 示例代码设计

继续扩展现有 `examples/java-basics` 模块，不新建平行基础模块：

```text
examples/java-basics/src/
├── main/java/io/github/javamastery/basics/
│   ├── arrays/
│   │   ├── ArrayStatistics.java
│   │   └── ArrayCopying.java
│   ├── strings/
│   │   ├── StringInspector.java
│   │   └── ReportBuilder.java
│   └── debugging/
│       └── DebuggingScenarios.java
└── test/java/io/github/javamastery/basics/
    ├── arrays/
    ├── strings/
    └── debugging/
```

实施使用上述类名，并遵守：

- 每个类只承担一个主要教学目标。
- 核心逻辑通过静态方法或小型实例方法暴露，便于 JUnit 验证。
- 调试场景的最终代码必须正确；错误版本使用课程片段、测试驱动步骤或显式练习分支展示，不能让默认 CI 保留已知缺陷。
- 不引入业务框架和额外运行时依赖。

## 6. 练习与答案设计

继续使用：

```text
exercises/java-basics/
├── starter/
└── solution/
```

新增 `statistics` 包承载成绩统计器：

```text
io.github.javamastery.exercises.basics.statistics
```

拆分为以下职责：

- `ScoreParser`：解析和校验成绩文本。
- `ScoreStatistics`：计算统计指标。
- `ScoreReport`：生成文本报告。

这些类只是按职责拆分代码，不承担完整面向对象建模教学。方法签名应保持简单，主要使用 `String`、`int[]`、`int` 和 `double`。

### 6.1 Starter

- 提供类、方法签名、JavaDoc、示例输入输出和可编译占位实现。
- 默认 Maven reactor 必须能够编译。
- 不提交故意失败并阻断全仓 CI 的测试。
- README 给出学习者启用练习测试的明确命令。

### 6.2 Solution

- 提供完整实现及正常、边界和非法输入测试。
- 测试覆盖解析、统计、报告格式和输入校验。
- 浮点断言使用容差，文本报告使用确定性格式。
- README 说明实现步骤、典型错误和建议断点位置。

## 8. 测试与验证

### 8.1 自动化测试

- 数组示例覆盖正常数组、单元素数组和空数组。
- 字符串示例覆盖正常文本、空文本、空白文本和内容比较。
- 成绩统计器覆盖正常输入、边界成绩、空输入、非法数字、缺失项和范围错误。
- 报告测试固定换行符、数字格式和字段顺序。
- Starter 保持可编译；Solution 的测试全部进入默认 CI。

### 8.2 人工验证

调试课程需要人工验证以下流程：

- IDEA 可以在示例和 JUnit 测试中命中断点。
- 单步执行能够观察循环下标与数组元素变化。
- Watches 能观察统计值的逐步变化。
- 调用栈能够定位解析、统计和报告生成之间的调用关系。
- 学习者能按照文档复现并修复至少三个典型错误。

IDEA 的人工验证结果不得替代 Maven、JUnit 和文档构建验证。

### 8.3 完成标准

v0.2.2 完成时必须满足：

- 3 篇新增课程达到 `learning-ready` 或 `verified`。
- 新增示例编译并通过 JUnit 5 测试。
- 成绩统计器 Starter 可以编译。
- 成绩统计器 Solution 测试全部通过。
- 全仓 `mvn -B clean verify` 通过。
- 文档内部链接检查通过。
- VitePress 构建通过。
- GitHub CI 通过。
- 合并后 GitHub Pages 成功部署并显示新增课程。

## 10. 非目标

v0.2.2 不包含：

- 类、对象、封装、继承和多态的系统教学。
- `List`、`Set`、`Map` 等集合框架。
- Stream API 和 Lambda 表达式。
- 文件读写、数据库或 Web 接口。
- 完整异常体系与自定义异常。
- 正则表达式系统课程。
- Unicode 代码点和国际化的深入处理。
- 图形界面或交互式命令行菜单。
- 英文课程翻译。

## 11. 风险控制

### 提前引入面向对象

成绩统计器只使用职责清晰的小类和基础方法，不讲领域模型、封装原则或继承体系。

### 调试课依赖特定 IDEA 版本

优先描述稳定的调试概念和操作名称；界面位置变化时给出搜索操作入口。所有最终结果仍由 Maven 和 JUnit 验证。

### 错误示例污染默认分支

故障代码只存在于课程片段或可控练习步骤中，仓库默认代码始终保持编译和测试通过。

### 字符串主题膨胀

只覆盖入门开发高频 API、内容比较和 `StringBuilder`。正则、编码、国际化和底层实现推迟到对应专题。

### 综合练习职责过重

解析、统计和报告生成保持独立，单个类和方法可单独理解、测试和调试。
