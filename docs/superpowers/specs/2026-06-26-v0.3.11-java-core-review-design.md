# v0.3.11 Java Core Review Design

## Goal

Close the Java Core stage with one learning-ready review lesson that helps learners integrate v0.3.0 through v0.3.10 concepts and use the existing object-oriented exercise set as the stage capstone.

## Scope

- Publish `docs/zh-CN/guide/02-java-core/12-java-core-review.md`.
- Frame the review around object modeling, invariants, inheritance, polymorphism, interfaces, exceptions, collections, generics, file I/O, date/time, streams, enums, and common standard library APIs.
- Reuse `exercises/object-oriented` as the Java Core capstone instead of adding a duplicate exercise module.
- Expand `exercises/object-oriented/README.md` with a stage-capstone introduction, knowledge-to-exercise mapping, recommended checkpoints, validation commands, and reflection prompts.
- Connect README, roadmap, the enums lesson, and VitePress navigation so the review becomes the natural next step after `11-enums-standard-library.md`.
- Update roadmap status text from "Java Core review is planned" to "v0.3.11 provides a learning-ready review and capstone path".
- Bump Maven project versions to `0.3.11-SNAPSHOT`.

## Out of Scope

- New Java production APIs beyond small documentation examples.
- A new `exercises/java-core-review` Maven module.
- Concurrency, JVM internals, database, web, Spring Boot, or v0.4 content.
- English documentation parity.
- A release tag or GitHub release.

## Design Notes

The review lesson should not repeat every chapter. It should help learners connect ideas across chapters: how encapsulation protects invariants, how polymorphism and interfaces support extension, how collections and streams shape data workflows, how exceptions and file I/O define failure boundaries, and how time, enums, `Optional`, `Objects`, `Comparator`, and `StringJoiner` appear in ordinary business code.

The capstone path should point at the existing course-learning model in `exercises/object-oriented`. That module already covers the stage's core concepts through starter skeletons, reference solution code, and JUnit tests. v0.3.11 should make this intent explicit and give learners a practical completion route.
